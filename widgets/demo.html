Freedback widgets

Run the stack first: docker compose up --build (feedback :8080, collection :8100). These widgets read aggregates from the collection server and publish to the feedback server. Two ways to authorize a write:

  • data-sign — the page generates a P-256 key (WebCrypto, kept in IndexedDB) and attaches a detached ES256 signature. No token needed; this is the federating identity. The key is extractable so you can password-back-it-up / move devices (see the identity card below). Needs a secure context (localhost counts).
  • data-token — an OAuth bearer (set FREEDBACK_OAUTH_TOKEN/_APP/_USER on the feedback server). The siloed, non-federating identity.

The widgets below use data-sign. Without either, a POST returns 401 (read-only still works).

Your signing identity (data-sign)

The federating identity is a P-256 key kept in this browser's IndexedDB. The issuer id below is your portable public identity. Back it up with a password so you can restore it on another device — or if this browser clears its storage. Rotating makes a new key; past signed feedback stays valid and a cross-signed link keeps the history attributable.

Lost your identity? (IndexedDB cleared)

If this browser cleared its storage, the in-browser key is gone and new feedback would sign under a new issuer id. To keep your old identity, click Restore (import) and paste a backup you exported earlier. Without a backup, the old key is unrecoverable — but any feedback already signed under it stays valid and verifiable forever.
