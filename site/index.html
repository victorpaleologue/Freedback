Freedback

A federated, open protocol for typed feedback on anything with a URI — carried as W3C Web Annotations.

▶ Try the live widgets Source on GitHub

See it in action

  • Widget showcase — the five drop-in widgets (stars, thumbs, scalar, comments, tags), live against an in-browser mock backend.
  • Annotate anything — attach feedback to a law, a Wikipedia article, an arbitrary web page (targets are just URIs).
  • Collect feedback (data collection) — a portable talk-feedback form you can export to CSV / JSON-LD.

What it is

Anyone can attach feedback — stars, scalar ratings, thumbs, comments, tags — to any resource with a URI, publish it to a server they choose, and have it discovered and aggregated across servers. No central gatekeeper. Feedback is signed with a portable key, so it is not locked inside the silo that collected it.

Protocol artifacts (stable URLs)

  • JSON-LD @context — /ns/context.jsonld
  • Vocabulary — /ns/freedback.ttl (rating subclasses of schema:Rating)
  • SHACL shapes — /ns/shapes.ttl (all validation lives here)
  • Validation profile — /profile/1 (dcterms:conformsTo)
  • Drop-in widget script — /widgets/freedback-widgets.js
